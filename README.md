# ✂️ EL-TARZY POS System

نظام نقاط بيع مخصص لإدارة محلات الترزي، مبني باستخدام Electron و Node.js.

## 🏗 المعمارية والتكنولوجيا
- **الإطار العام**: Electron (Main & Renderer process architecture).
- **قاعدة البيانات**: `better-sqlite3` (محلية بالكامل، لا تتطلب إنترنت).
- **واجهة المستخدم**: HTML/CSS/JS (Vanilla) بدون أطر عمل خارجية، لضمان السرعة القصوى.
- **واتساب**: `whatsapp-web.js` مدمج مع متصفح Edge/Chrome المحلي لتجنب مشاكل Puppeteer في الحزمة المجمعة (`asar`).
- **التشفير والتفعيل**: يعتمد على `crypto` لتوليد مفتاح تفعيل مرتبط بـ Hardware ID للحماية من القرصنة.

## 📂 هيكل المشروع
- `main.js`: نقطة الدخول (Entry point)، يدير دورة حياة التطبيق، النوافذ، والواتساب.
- `preload.js`: الجسر (Bridge) بين الـ Main Process والـ Renderer Process (يحمي من استغلال الـ IPC).
- `database/`: 
  - `db.js`: الملف الأساسي للتعامل مع قاعدة البيانات وتخزين الإعدادات والفواتير.
  - `schema.sql`: هيكل جداول قاعدة البيانات.
  - `seed.js`: بيانات البداية (الإعدادات الافتراضية، حساب الأدمن).
- `whatsapp/`:
  - `whatsapp-service.js`: يدير جلسة الواتساب ويبحث عن متصفح مدعوم.
- `renderer/`: يحتوي على ملفات الواجهة (HTML) مقسمة لصفحات (فاتورة، موارد بشرية، مرتجعات، تقارير... إلخ).
- `renderer/js/`: يحتوي على أكواد الجافاسكريبت المفصولة عن الـ HTML.
  - `shared/common.js`: دوال مشتركة لكل الواجهات (`showToast`, `fmt`, `goBack`... إلخ).

## 🚀 كيفية التشغيل والترجمة
1. `npm install`
2. `npm run postinstall` (لإعادة بناء `better-sqlite3` حسب معمارية الجهاز)
3. `npm start` للتشغيل في بيئة التطوير.
4. `npm run build` لتصدير البرنامج كملف تنفيذي (`El-Tarzy.exe`).

## 🛡 الأمان والصلاحيات
- **كاشير (Cashier)**: مقيد بصلاحيات البيع فقط. لا يمكنه مسح فواتير، تعديل الأسعار، أو رؤية الرواتب والإحصائيات.
- **مدير (Admin)**: لديه صلاحية وصول كاملة لكل وحدات النظام.
- يتم التحقق من الصلاحيات على مستوى الـ `db.js` (Backend) لمنع التجاوز عن طريق واجهة المستخدم.

## 🔄 التحديثات والـ Migrations
يتم ترقية هيكل قاعدة البيانات برمجياً من خلال دالة `migrateSchema` في `db.js` التي تستخدم نظام إصدارات لضمان التوافق مع النسخ الأقدم.
